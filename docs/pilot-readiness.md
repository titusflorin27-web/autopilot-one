# Pilot Readiness Checklist

## Objective

Prepare the first private pilot for Autopilot One.

A pilot is ready when the product can support one real business through the complete flow:

Business DNA → Knowledge Base → Reception AI → Website Widget → Inbox → Notifications → Plans and Usage.

## Pilot candidate requirements

Choose a pilot business that has:

- a clear website
- repeat customer questions
- at least one service or product offer
- willingness to test an AI reception workflow
- one internal owner who can review Inbox and Notifications

## Pre-pilot setup

### Organization

- Create organization workspace.
- Confirm organization name and slug.
- Confirm language and timezone.

### Business DNA

- Add company summary.
- Add services or products.
- Add rules and boundaries.
- Add FAQ.
- Set tone of voice.
- Add objectives.

### Knowledge Base

- Add at least one source.
- Prefer real website FAQ, service page or company document.
- Verify search returns useful chunks.

### Widget

- Enable widget.
- Set title.
- Set primary color.
- Set position.
- Generate widget token.
- Add allowed origin for test site.
- Copy install snippet.

### Public test

- Load widget on test page.
- Send a realistic customer message.
- Confirm response appears.
- Confirm conversation appears in Inbox.
- Confirm widget analytics shows activity.

### Operator flow

- Open Inbox.
- Review conversation.
- Add human reply.
- Move conversation through open, handoff and closed states.
- Review Notifications.
- Review Launch Checklist.

## Pilot success metrics

Track these during the first pilot week:

- widget loads
- widget opens
- public messages
- conversations created
- lead count
- human handoffs
- tasks created
- operator replies
- closed conversations

## Daily pilot routine

For each pilot day:

1. Check Widget Analytics.
2. Check Inbox.
3. Check Notifications.
4. Review new leads.
5. Close resolved conversations.
6. Note missing Knowledge Base content.
7. Update Business DNA or Knowledge Base if needed.

## Pilot exit criteria

The pilot is successful if:

- widget remains stable
- Reception AI answers common questions usefully
- human handoff works
- operator can manage conversations in Inbox
- at least one useful lead or task is created
- business owner understands the value proposition

## Follow-up after pilot

After the pilot:

- collect customer feedback
- document missing features
- identify confusing UI steps
- review usage and limits
- decide next build priority
